# Теоретический материал

## Разработка скриптов на BASH

### Цель работы
Получить практические навыки программирования на bash, освоить практические приемы работы с процессами в Linux, закрепить материалы лекций по архитектуре CPU.

### Необходимо:
- ОС Linux на реальной или виртуальной машине

### Краткие теоретические сведения

Традиционным средством автоматизации для операционных систем является использование скриптовых языков. Скрипт представляет собой текстовый файл с кодом программы, который подается на исполнение в бинарный файл командного интерпретатора. То есть не происходит компиляция кода в бинарный исполняемый файл. 

Командные интерпретаторы часто выступают в роли основного **shell** – программной среды, предоставляющей интерфейс управления ОС для пользователя. 

Для UNIX-подобных систем одним из популярных (но не единственных) командных интерпретаторов является `bash` - **Bourne-Again SHell** («перерожденный шелл» старый UNIX шелл - `sh`). В Linux системах он располагается по пути `/bin/bash` и является шеллом по умолчанию для пользователей. Bash позволяет запускать команды (внешние бинарные файлы) и использовать встроенные в сам bash команды.

## Основные концепции языка bash

1. Все переменные имеют один тип — строка. Интерпретатор определяет, что строка содержит число, по контексту программы.
2. Объявление переменной:
   ```bash
   VARIABLE_NAME="value"
   VARIABLE2_NAME=5
   ```
3. Обращение к переменным делается так:
    ```bash
    $VARIABLE_NAME
    ${VARIABLE_NAME}
    ```

## Арифметические операции в bash

Для арифметических операций используется конструкция `$(( ))`. Вот классический пример:

```bash
  #!/bin/bash
  # Задание значений переменных
  num1=10
  num2=5
  # Выполнение арифметических операций
  sum=$((num1 + num2))
  difference=$((num1 - num2))
  product=$((num1 * num2))
  quotient=$((num1 / num2))
  remainder=$((num1 % num2))
  # Вывод результатов
  echo "Сумма: $sum"
  echo "Разность: $difference"
  echo "Произведение: $product"
  echo "Частное: $quotient"
  echo "Остаток от деления: $remainder"
```

Учитывается порядок операций и скобочки:
```bash
result=$(( (5 + 3) * 2 ))
echo $result # Вывод: 16
```

Есть инкремент и декремент (но с хитринкой):
```bash
(( count++ )) # Инкремент
(( count-- )) # Декремент
```

## Использование bc для вещественных вычислений

<Callout>
Учтите, что все операции в bash целочисленные.
</Callout>

Если нужно считать что-то сложное, то используйте утилиту [bc](https://www.gnu.org/software/bc/manual/html_mono/bc.html).

Например, вот деление с точностью до 4-го знака после запятой.
```bash
echo 'scale = {{4}}; {{5 / 3}}' | bcф
```

# Порядок выполнения работы

## Часть 1. Получение информации о процессах

1. Убедитесь, что в настройках виртуальной машины выделено несколько ядер CPU.
2. Запустите систему и авторизуйтесь с правами `root`.
3. На первой консоли установите и запустите утилиту `htop` (можно использовать старый вариант `top`).
4. На второй консоли запустите команду:
   ```bash
   ping 127.0.0.1
   ```
5. На третьей консоли с помощью команды `ps` определите для процесса `ping` его:
   - **PID**
   - **PPID**
   - **команду запуска**
   - **пользователя**, от имени которого работает процесс
   - **параметр nice**
   - **логический процессор**, на котором он выполняется.
6. Определите те же данные с помощью утилиты `htop` на первой консоли и завершите средствами `htop` процесс `ping`.
7. На третьей консоли завершите процесс `htop` с помощью команды `kill`.

## Часть 2. Управление запуском процесса

1. С помощью утилиты `taskset` запустите процесс на выбранном вами ядре.
2. Убедитесь, что процесс стартовал на нужном ядре.
3. На второй консоли с помощью утилиты `taskset` переместите процесс `htop` на другое ядро.
4. Убедитесь, что перемещение произошло.
5. Напишите конвейер команд, выводящий номера ядер и количество процессов, запущенных на каждом из них.


## Часть 3. Разработка скрипта расчета производительности

1. Разработайте скрипт, который определяет производительность вашего CPU в FLOPS (*Floating Point Operations Per Second* или «операции с плавающей запятой в секунду»). Этот показатель используется для измерения вычислительной мощности компьютеров, когда требуется выполнять сложные математические операции с высокой точностью. Операции с плавающей запятой требуют больше ресурсов по сравнению с операциями с целыми числами. Измеряя производительность во FLOPS, определяют, сколько таких операций процессор может выполнить за одну секунду.
2. Создайте скрипт, который выполняет не менее миллиона операций умножения двух одинаковых нецелых чисел с точностью до 10 знаков после запятой и, определив затраченное время в секундах, вычисляет значение FLOPS. Для расчетов используйте `bc`. Подавите лишний консольный вывод, перенаправив его в специальный «бесконечный» файл `/dev/null`. Вычисление следует оформить в виде функции. Время сохраните в переменные перед и после запуска функции.


## Часть 4. Управление приоритетом процессов

1. Создайте два скрипта:
   - Первый скрипт выполняет математические вычисления в бесконечном цикле.
   - Второй скрипт читает файл объемом 1 Гб, перенаправляет вывод в другой файл, удаляет исходный файл и повторяет процедуру с его копией. Создайте файл с помощью утилиты `fallocate`.
2. Запустите оба скрипта в фоновом режиме (используйте оператор `&`, указанный через пробел после вызова скрипта). Один из процессов запустите с высоким приоритетом (отрицательное значение `nice`), другой — с низким приоритетом (положительное значение `nice`).
3. Проанализируйте, как изменяется процессорное время для каждого процесса после изменения их приоритетов.
4. Создайте искусственную нагрузку на систему с помощью утилиты `stress`, задействовав одно ядро. Оцените, как это влияет на ранее запущенные процессы.
5. Попробуйте изменить приоритеты или привязку к ядрам для улучшения производительности.

### Цель и задачи

* Опробовать типовой стек команд **GNS3** на практике.

### Ход работы

<Subtitle>
  ## Часть 1 — Развёртывание инфраструктуры
</Subtitle>

1. Запустите лабораторную работу, нажав справа на кнопку <Button className="pointer-events-none">Запустить</Button>.

2. Разместите оборудование.

    * <span className='flex items-center gap-2'>2 × компьютера (PC) <GNS3Node node="vpcs" /></span>

3. **Соедините устройства** витой парой (Ethernet0/0, FastEthernet0/1 и т. д.) в соответствии с диаграммой.

4. **Назначьте IP-адреса** компьютерам:

Конфигурация для PC1:

```bash
ip 192.168.1.2 255.255.255.0 192.168.1.1
```

Конфигурация для PC2:

```bash
ip 192.168.1.3 255.255.255.0 192.168.1.1
```

5. **Проверьте доступность узлов**:
```bash
PC1> ping 192.168.1.3

# Пример ответа
84 bytes from 192.168.1.3 icmp_seq=1 ttl=64 time=0.624 ms
84 bytes from 192.168.1.3 icmp_seq=2 ttl=64 time=0.796 ms
```

6. Запустите автоматическую проверку лабораторной работы, нажав на кнопку <Button className="pointer-events-none">Отправить</Button> на странице управления лабораторной работой

После успешной проверки в таблице "Проверки" должна появиться запись с результатами проверки.
  - Проект открыт
  - Узлы проекта запущены
  - Ключевые узлы обнаружены
  - Параметры узлов проекта получены
  - Извлечены IP адреса ключевых узлов
  - Проверка доступности (PC1) -> (192.168.1.3)

# Практика и безопасность в сетях

Знание теории – это замечательно, но умение применять её на практике ещё важнее. В данной главе мы рассмотрим базовые команды и инструменты для настройки сети, а также основные принципы защиты сетевой инфраструктуры.

## Базовые инструменты настройки и диагностики

### ifconfig / ipconfig / ip

В Linux-системах команда `ifconfig` (устаревшая) или `ip` (современная альтернатива) помогает просматривать и настраивать параметры сетевых интерфейсов. В Windows используется команда `ipconfig`.

```bash title="Пример вывода ip в Linux"
// Посмотреть информацию о сетевых интерфейсах
ip addr show

// Настроить IP-адрес (пример)
sudo ip addr add 192.168.1.10/24 dev eth0
```

### netstat / ss

- `netstat` (старый вариант) или `ss` (более современный инструмент) позволяют просматривать соединения, таблицы маршрутизации и статистику сетевых протоколов.

```bash title="Пример вывода ss"
// Отобразить все сетевые соединения
ss -tuln
```

<Callout>
  Команда `ss` работает быстрее и даёт более детальную информацию о текущих
  сетевых соединениях по сравнению с `netstat`.
</Callout>

## Мониторинг и безопасность

1. **Firewall (Межсетевой экран)**: Позволяет фильтровать входящие и исходящие пакеты, защищая сеть от несанкционированного доступа.

   - Примеры: `iptables`, `ufw` (Linux), встроенный Firewall в Windows.

2. **IDS и IPS (Intrusion Detection/Prevention System)**: Системы обнаружения и предотвращения вторжений анализируют трафик и выявляют подозрительную активность.

   - Пример: `Snort`, `Suricata`.

3. **VPN (Virtual Private Network)**: Обеспечивает защищённый канал связи поверх небезопасной сети (Интернет).
   - Примеры: OpenVPN, WireGuard, IPSec.

### Практический пример использования iptables

```bash title="Пример базовой конфигурации iptables"
# Запретить весь входящий трафик по умолчанию
sudo iptables -P INPUT DROP

# Разрешить установленные входящие соединения и локальные запросы
sudo iptables -A INPUT -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
sudo iptables -A INPUT -i lo -j ACCEPT

# Разрешить SSH (порт 22)
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
```
